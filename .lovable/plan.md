## Phase 2 — Responder Android App

A minimal standalone Capacitor Android app (separate from the Glance web app) that a responder installs once, pairs via QR, and then silently streams GPS to `dispatch-ping` whenever an alert is assigned to them.

### Repo layout
Create a new sibling folder in this repo (no impact on the operator web app):

```text
apps/responder-android/
  package.json
  capacitor.config.ts
  index.html            # single-screen shell (pair / status / manual controls)
  src/
    main.ts             # bootstraps app, pairing, foreground service, ping loop
    api.ts              # dispatch-ping + dispatch-state calls
    storage.ts          # Preferences plugin wrapper (token, endpoint)
    pairing.ts          # QR scan → parse token payload → store
    tracker.ts          # geolocation watch + background heartbeat
  android/              # generated by `npx cap add android` (committed after first sync)
```

Operator web app under `src/` is untouched.

### Behaviour
1. **First launch** — screen shows "Pair device". Responder taps → camera opens (ML Kit barcode via `@capacitor-mlkit/barcode-scanning`) → scans QR from the Responders page → app stores `{ token, endpoint }` in `Preferences` and registers.
2. **Idle** — app sits in foreground service with a persistent notification "Glance Responder — ready". No location sent while there is no active dispatch.
3. **Active dispatch** — app polls `dispatch-state` every 15 s. If any dispatch is `pending` or `en_route` for this device, it starts `Geolocation.watchPosition` (high accuracy, 10 s interval) and POSTs each fix to `dispatch-ping`. Server-side geofence promotes `en_route` → `arrived`; when status becomes `completed`/`cancelled`, tracking stops.
4. **Manual controls** — three buttons: Acknowledge / Arrived / Complete → call `dispatch-state`. Handy if GPS geofence misses.
5. **Auto-start on boot** — `RECEIVE_BOOT_COMPLETED` broadcast receiver re-launches the foreground service so a rebooted phone resumes silently.

### Server side
Add a lightweight `dispatch-poll` edge function (or reuse `dispatch-state` with a `GET`) that returns the current open dispatch for the calling device token. This is what the phone hits every 15 s while idle.

### Permissions (AndroidManifest additions)
- `ACCESS_FINE_LOCATION`, `ACCESS_COARSE_LOCATION`, `ACCESS_BACKGROUND_LOCATION`
- `FOREGROUND_SERVICE`, `FOREGROUND_SERVICE_LOCATION`
- `POST_NOTIFICATIONS`
- `INTERNET`, `RECEIVE_BOOT_COMPLETED`, `WAKE_LOCK`
- Camera (pairing QR only)

### Dependencies
`@capacitor/core`, `@capacitor/cli`, `@capacitor/android`, `@capacitor/geolocation`, `@capacitor/preferences`, `@capacitor/app`, `@capacitor-community/background-geolocation` (for reliable background fixes), `@capacitor-mlkit/barcode-scanning`.
